# How-to: 新しいテナント（飲食店）を追加する

Fleximo では1つのデプロイに複数のテナント（飲食店）を相乗りさせます。
このガイドは、運用者が新しいテナントを1つ追加する手順を示します。

## ゴール

- 新しいテナントが顧客側の検索に表示される
- テナント管理者アカウントでログインし、メニューを登録できる

## 前提

- プラットフォーム管理者のアカウントを持っていること
- テナント側の担当者の情報（店舗名、住所、代表者、連絡先）が揃っていること
- fincode 加盟店契約が **「プラットフォームタイプ」** であること（テナント別の `fincode_shop_id` 切り替えが成立する前提）。詳細は [fincode 決済の設定](./configure-fincode.md) を参照

## 手順

### 1. テナントレコードを作る

Artisan コマンドで作成します。

```bash
php artisan tenant:create \
  --name="○○食堂" \
  --slug=maru-maru-shokudo \
  --admin-email=owner@example.com
```

実行すると、テナント管理者アカウントの初期パスワードが標準出力に出ます。
このパスワードは一度しか表示されないので、安全な方法で店舗側に連携してください。

### 2. 営業情報を登録する

テナント管理者でログインし、「設定 → 店舗情報」から以下を入力します。

- 店舗名（表示用）
- 住所・電話番号
- 営業時間
- 受取方法（MVP ではテイクアウトのみ）

### 3. メニューを登録する

「メニュー → 新規作成」から商品を1件追加します。
画像は `storage/app/public/tenants/{tenant_id}/menus/` に保存されます。

### 4. 決済設定を確認する

テナントが自前の fincode アカウントを使う場合は、「設定 → 決済」でテナント固有のキーを入力します。
プラットフォーム共通の fincode を使う場合はデフォルトのままで構いません。

### 5. 公開

テナント管理画面の「ステータス」を `公開` に切り替えると、顧客側の店舗検索結果に表示されます。

## 削除・一時停止

テナントを一時停止する場合は削除せず、`ステータス: 非公開` に変更してください。
データを残したまま検索結果から外せます。削除すると注文履歴との整合性が崩れる可能性があります。
